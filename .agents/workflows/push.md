# 🚀 Push Work Workflow

## 📋 Instructions
เมื่อได้รับคำสั่ง `/push` ให้ดำเนินการดังนี้:

1. **Summarize Changes**: วิเคราะห์การเปลี่ยนแปลงล่าสุดใน codebase
2. **Commit**: 
    - รัน `git add .`
    - รัน `git commit -m "<summary of changes>"` (ใช้ภาษาอังกฤษที่สื่อความหมายชัดเจน)
3. **Push**: รัน `git push`
4. **Save Context**: ทำการรันคำสั่ง `/save` (หรือทำตามขั้นตอนใน `complete-work.md`) เพื่ออัปเดตสถานะงานล่าสุด
5. **Notify**: แจ้งผู้ใช้ว่าทำการ Push และ Save สถานะงานเรียบร้อยแล้ว
